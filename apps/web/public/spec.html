agent-feed v0

A signed, machine-readable announcement layer for changes to agent-facing surfaces. robots.txt + DKIM + RSS, designed for autonomous agents.

This document specifies agent-feed, a protocol by which an origin signs and publishes a stream of changes to its machine-readable agent surface. agent-feed is protocol-agnostic: it composes beneath MCP, A2A, ANP, and OpenAPI, and announces changes orthogonal to what each of those standards describes. The reader contract is specified before the producer schema; this is deliberate.

§1Consumer behavior

A consumer (an autonomous agent, an agent operator's CI, an aggregator) MUST fetch /.well-known/agent-feed.xml over HTTPS and MUST verify the Ed25519 signature against the origin's did:web document before treating any entry as authoritative. Unverified entries MUST NOT be used to invalidate a pin.

A consumer SHOULD additionally fetch /.well-known/agent-card.json for the current snapshot, but MUST distinguish snapshot from stream when surfacing facts to humans or downstream agents.

§2Trust planes

This specification distinguishes two trust planes that MUST NOT be conflated:

  • Signed — first-party, Ed25519-attested, did:web-rooted. Testimony.
  • Observed — third-party, possibly stale, always tagged UNSIGNED in any rendering. A photograph.

Implementations MUST NOT render observed entries in a way that suggests signed-grade trust, including but not limited to: omitting the UNSIGNED tag, color-coding observed and signed identically, or sorting them in a single mixed list.

§3Two artifacts, two URLs

An origin MUST publish two distinct well-known artifacts:

§3.1 — /.well-known/agent-card.json

The current state. Versioned. Fetched on first contact. Answers what is true now.

§3.2 — /.well-known/agent-feed.xml

The history. Append-only. Answers how did we get here, and what's ahead. 

<feed xmlns="https://agent-feed.dev/v0"
      origin="did:web:example.com">
  <entry type="deprecated" since="2026-04-12">
    <ref>/v1/orders</ref>
    <successor>/v2/orders</successor>
    <sunset>2026-10-01</sunset>
  </entry>
  <sig alg="Ed25519" key="did:web:example.com#k0">…</sig>
</feed>

§5Signing

Every entry MUST be signed with an Ed25519 key referenced from the origin's did:web document. Key rotation is described in §5.2 of the full spec; this stub omits it.

§9Conformance

Requirement keywords (MUST, MUST NOT, SHOULD, SHOULD NOT) follow RFC 2119 / 8174 when, and only when, they appear in capitals.
